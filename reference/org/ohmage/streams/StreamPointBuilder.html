public void write (StreamWriter writer)

Uses a StreamWriter to send data to ohmage. The StreamWriter connects to a remote service with a oneway interface.

This method is useful when you have a lot of points to send and you have a clear start and end. If the StreamWriter is not connected yet, this method will automatically buffer the point and try to connect. After the data is sent, the writer should be closed to disconnect from the remote service.

This method returns slower than the async calls, but generally the data is saved to ohmage faster overall.

Parameters
writer a StreamWriter to send points.
Throws
RemoteException can be thrown if there was a problem sending the data to ohmage.

public void write (ContentResolver resolver)

Send this point to ohmage.

If you have many points to send within a few milliseconds of each other, you will get better performance by calling write(StreamWriter) or writeAsync(AsyncBulkInsertHandler).

This call will block until the point has been saved in ohmage.

Parameters
resolver a ContentResolver that is used to save the point

public void writeAsync (AsyncQueryHandler handler, int token, Object cookie)

Asynchronously send this point to ohmage.

If you have many points to send within a few milliseconds of each other, you will get better performance by calling write(StreamWriter) or writeAsync(AsyncBulkInsertHandler).

This call returns without waiting for ohmage to save the point. Implement onInsertComplete(int, Object, Uri) for the handler provided to get callbacks whenever a point is saved to ohmage.

Before sending data to ohmage, the stream should check that the content provider exists by calling checkContentProviderExists(ContentResolver)

Parameters
handler an AsyncQueryHandler to send the point
token A token passed into onInsertComplete(int, Object, Uri) to identify this write operation.
cookie An object that gets passed into onInsertComplete(int, Object, Uri)

public void writeAsync (AsyncBulkInsertHandler handler)

Asynchronously queue this point to be sent to ohmage in a batch.

If there is more time than the flush delay of the AsyncBulkInsertHandler between points you will get better performance by calling write(ContentResolver) or writeAsync(AsyncQueryHandler, int, Object).

This call returns without waiting for ohmage to save points. Implement onBulkInsertComplete(int) for the handler provided to get callbacks whenever batches of points are saved to ohmage.

Before sending data to ohmage, the stream should check that the content provider exists by calling checkContentProviderExists(ContentResolver)

Parameters
handler an AsyncBulkInsertHandler to send batches of points
